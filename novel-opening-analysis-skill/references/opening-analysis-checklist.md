# Opening Analysis Checklist

Use this checklist before declaring a first-three-chapter analysis complete.

## Core Pass Conditions

- [ ] `README.md` exists and correctly describes the opening-analysis workspace
- [ ] latest `工作状态-YYYY-MM-DD.md` exists and reflects the real state
- [ ] `<小说名>-黄金前三章总判断.md` contains a direct opening judgment
- [ ] `<小说名>-第一章拆解.md` is no longer placeholder text
- [ ] `<小说名>-第二章拆解.md` is no longer placeholder text
- [ ] `<小说名>-第三章拆解.md` is no longer placeholder text
- [ ] `<小说名>-开篇钩子与读者承诺.md` clearly explains hook and promise
- [ ] `<小说名>-开篇问题与修改建议.md` clearly explains problems and revision priorities

## Chapter-Level Questions

For each chapter, confirm that the analysis answers:

- What is this chapter structurally doing?
- What hook or pressure exists here?
- What does the protagonist want, fear, or fail to get?
- What information is being released?
- Does the chapter ending create forward pull?
- What should be strengthened, trimmed, delayed, or advanced?

## Total-Judgment Questions

Confirm that the total-judgment file answers:

- Is the opening hook strong enough?
- Is the protagonist clear fast enough?
- Does meaningful conflict start in time?
- Is the worldbuilding helping or slowing the opening?
- Would a reader likely continue after chapter three?

## Stop Rule

Do not declare completion just because the file set exists.

Only stop when both are true:

- `开篇抓力已明确`
- `前三章结构已拆清`
