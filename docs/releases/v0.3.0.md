# Release v0.3.0

Release date: 2026-05-01

## Summary

`v0.3.0` is the first public-facing productized release of `novel-workspace-pipeline`.

This version turns the repository from a private mixed workspace into a cleaner public workflow package with:

- portable workflow scripts
- product-style documentation
- a permissive open-source license
- public synthetic demo workspaces
- basic repository governance documents

## Highlights

### 1. Public workflow packaging

- split the reusable workflow out of the larger mixed-content workspace
- kept the five analysis layers plus orchestrator
- removed dependence on machine-specific absolute paths in key init flows

### 2. Better repository entrypoint

- rewrote the top-level `README.md`
- added `CONTRIBUTING.md`
- added supporting docs under `docs/`
- made the repository easier to understand as a product instead of a personal archive

### 3. Public demo workspaces

- added `fixtures/demo-outline-repair`
- added `fixtures/demo-opening-missing`
- connected default regression to those public fixtures

Current public regression result:

```text
workspace regression: 2/2 passed
```

### 4. Governance basics

- added `LICENSE` using `MIT`
- added `SECURITY.md`
- added `CHANGELOG.md`

## Included Components

- `novel-workspace-orchestrator-skill`
- `novel-chapter-distillation-skill`
- `novel-opening-analysis-skill`
- `novel-protagonist-encyclopedia-skill`
- `novel-outline-analysis-skill`
- `novel-highlight-scenes-analysis-skill`
- `text2markdown`
- `novel-character-cards`

## Known Boundaries

- this is a workflow system, not a fully autonomous writing agent
- the repository intentionally excludes private real-novel workspaces
- `repair-existing` is still stronger at structural repair than semantic content rewriting

## Upgrade Notes

If you were using an earlier internal copy of this project:

- prefer repository-relative paths
- keep raw source texts and large generated outputs out of the public repo
- use the synthetic fixtures as the default public regression baseline
