# Chapter Distillation Checklist

Use this checklist before declaring a chapter-distillation package complete.

## Core Pass Conditions

- [ ] `README.md` exists and correctly describes the chapter-distillation workspace
- [ ] latest `工作状态-YYYY-MM-DD.md` exists and reflects the real state
- [ ] `chapter-distillation-manifest.json` exists and lists detected chapters
- [ ] `<小说名>-章节蒸馏骨架.md` includes every detected chapter and is no longer placeholder text
- [ ] `<小说名>-阶段骨架与换挡草图.md` explains likely stage changes
- [ ] `<小说名>-校准与验证锚点.md` explains later verification anchors

## Chapter-Level Questions

For each chapter, confirm that the distillation answers:

- What actually moves here?
- What is the protagonist or viewpoint state?
- What new information appears?
- What relationship or situation changes?
- What is this chapter structurally doing?
- What hook or pressure carries forward?

## Whole-Package Questions

Confirm that the package answers:

- Has every chapter been distilled?
- Where does the novel first promise its main direction?
- Where are the major gear-shift points?
- Which chapters should later skills revisit to avoid drift?
- Is the skeleton detailed enough to support mid-project recalibration?

## Stop Rule

Do not declare completion just because the chapter file exists.

Only stop when both are true:

- `章节骨架已形成`
- `校准锚点已可用`
