# Highlight Scenes Analysis Checklist

Use this checklist before declaring a whole-book highlight-analysis package complete.

## Core Pass Conditions

- [ ] `README.md` exists and correctly describes the highlight-analysis workspace
- [ ] latest `工作状态-YYYY-MM-DD.md` exists and reflects the real state
- [ ] `<小说名>-最吸引人的十个剧情细节总表.md` contains a direct Top 10 judgment
- [ ] `<小说名>-剧情吸引力机制分析.md` is no longer placeholder text
- [ ] `<小说名>-Top10细节逐条拆解.md` is no longer placeholder text
- [ ] `<小说名>-高光桥段分布与节奏判断.md` clearly explains highlight distribution
- [ ] `<小说名>-最强爽点痛点悬念点总结.md` clearly explains what kind of reader pleasure dominates
- [ ] `<小说名>-剧情高光改造建议.md` clearly explains what should be protected or repaired

## Scene-Level Questions

For each selected highlight scene, confirm that the analysis answers:

- What exactly happens here?
- What setup exists before it?
- Why does it explode at this point?
- What kind of reader pleasure does it trigger?
- What changes because of it?
- Is it memorable enough to retell?

## Whole-Package Questions

Confirm that the package answers:

- What are the strongest 10 scenes in the whole book?
- Which attraction engine dominates this novel?
- Are the highlights well distributed across the book?
- Does the middle book keep enough energy?
- What should be amplified, advanced, delayed, merged, or compressed?

## Stop Rule

Do not declare completion just because the file set exists.

Only stop when both are true:

- `高光桥段已明确`
- `剧情吸引力机制已拆清`
